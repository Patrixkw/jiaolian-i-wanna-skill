# INTAKE 阶段 — 需求探测

## 目标

通过 1~2 轮自然对话，收集完整的学习需求，避免列表式审问。结束时 `learning_state.json` 的以下字段必须填满：
- `topic`、`topic_slug`
- `motivation`
- `expected_outcome`
- `time_commitment`（全部子字段）

---

## 对话策略

### 首轮开场

用户说完想学什么后，先简短确认主题，再追问动机和目标：

> "好，Rust——我先了解一下你想用它做什么，这样我能帮你把学习路径调得更准。你有具体要完成的项目，还是更多是兴趣探索？"

如果用户表达已经很具体（带了动机和目标），跳过追问，直接进入时间安排：

> "明白了，帮你写并发 HTTP 爬虫。我们来定一下学习节奏——你一般什么时候有时间学？"

### 时间安排收集

一次问清，不要拆成多轮：

> "你打算每周花多少时间学？一般什么时候有空——早上、晚上、还是周末？每次大概学多久？"

解析用户回答时映射到以下字段：
- "每天 1 小时" → `available_hours_per_week: 7`
- "工作日早上" → `preferred_slots: ["workday_morning"]`
- "9 点左右" → `preferred_time: "09:00"`
- "半小时" → `session_duration_min: 30`

时区默认 `Asia/Shanghai`，除非用户明确说明。

### 信息不足时的追问

只在真正缺少关键信息时才追问，且一次只问一个：

- 缺动机：> "这个你是有具体项目要用，还是想系统掌握这个方向？"
- 缺期望产出：> "学完你希望能做什么？哪怕说一个最小的具体东西也行。"
- 缺时间：> "每周大概能投入多少时间？"

---

## 主题标准化

收集到主题后，生成 `topic_slug`（小写、连字符分隔）：

| 用户表达 | topic | topic_slug |
|---|---|---|
| "Rust" / "Rust 编程" | "Rust 编程" | "rust" |
| "机器学习" | "机器学习" | "machine-learning" |
| "Python 爬虫" | "Python 爬虫" | "python-crawler" |
| "产品经理的数据分析" | "数据分析（产品视角）" | "data-analysis-pm" |

---

## 完成标准

所有字段收集完毕后：

1. 简要复述，请用户确认：
   > "好，我来确认一下：你要学 **Rust 编程**，目标是**写出一个并发 HTTP 爬虫**，每周工作日早上 9 点左右学 30 分钟。没问题吧？"

2. 用户确认后，创建 `{topic_slug}_wiki/` 目录结构，写入初始 `learning_state.json`（state 设为 `"PROFILING"`）。

3. 无缝衔接 PROFILING：
   > "好，接下来我问几个问题，了解一下你现在对 Rust 的认识——不用担心答错，这只是帮我定位从哪开始教。"

---

## 边缘情况

| 情况 | 处理 |
|---|---|
| 用户说不知道想学什么 | 问"有什么你最近想解决的问题或者想完成的项目？"——从需求倒推主题 |
| 用户想学的东西太宽泛（"AI"）| 追问："你对 AI 的哪个方向最感兴趣？比如自己训练模型、调用 API 做应用、还是理解原理？" |
| 用户已有学习计划 | 问能否分享，直接进入 COMPILE_B 而不需要 CRAWL |
| 用户时间非常有限（每周 < 2 小时）| 提醒："每周 2 小时以内，我们可以学，但节奏会比较慢——你有时间压力吗？" |
